# SMTP / Nodemailer (초대 메일 — POST /api/users)
# =================================================================
# Nodemailer는 "API 키 1개"가 아니라 SMTP 접속 정보(호스트·계정·비밀번호)를 씁니다.
# 아래 중 하나의 메일 서비스에서 SMTP Relay(또는 앱 비밀번호)를 발급받아 .env에 넣습니다.
# 상세: docs/plans/02_user-invite-profiles-plan.md (초대 방식 변경 시 SMTP 절 참고)
#
# 공통 — .env 변수 (Gmail 예시)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@gmail.com
SMTP_PASSWORD=
# ↑ https://myaccount.google.com/apppasswords — 2단계 인증 후 앱 비밀번호 16자 (공백 제거)
SMTP_FROM=WakeOne <your@gmail.com>
# SMTP_SECURE=false  # Gmail 587은 보통 false
# SMTP_SECURE=true  # 포트 465(SSL)일 때만 보통 true
#
# --- 어디서 무엇을 가져오는지 (개발자용) ---
#
# | 서비스 | 가입/콘솔 | 가져올 값 → .env 매핑 |
# |--------|-----------|------------------------|
# | Gmail (개인/Workspace) | https://myaccount.google.com/apppasswords | 2단계 인증 후 "앱 비밀번호" 생성 → SMTP_PASSWORD. SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=본인@gmail.com |
# | Brevo (구 Sendinblue) | https://app.brevo.com → SMTP & API → SMTP | SMTP 서버/포트/로그인 표 그대로 → HOST/PORT/USER/PASS. 발신: SMTP_FROM에 인증된 발신 주소 |
# | SendGrid | https://app.sendgrid.com → Settings → API Keys (SMTP용) | "Create API Key" 또는 SMTP Relay: SMTP_USER=apikey, SMTP_PASSWORD=생성한 API Key, HOST=smtp.sendgrid.net PORT=587 |
# | Mailgun | https://app.mailgun.com → Sending → Domain → SMTP credentials | 표시된 host/port/user/password |
# | AWS SES | https://console.aws.amazon.com/ses → SMTP settings | Create SMTP credentials → HOST/PORT/USER/PASS (리전별 endpoint) |
# | Microsoft 365 | Azure/Entra 또는 M365 관리센터 | SMTP AUTH 허용 필요. HOST=smtp.office365.com PORT=587 USER=사서함 이메일 PASS=앱 비밀번호 |
#
# 네이버/다음 일반 메일: 대량·앱 발송 SMTP가 제한되는 경우가 많음 → Brevo/SendGrid/Gmail Workspace 권장.
# 로컬 테스트: https://ethereal.email (가짜 SMTP, 수신함 웹에서 확인) — 운영 금지.
